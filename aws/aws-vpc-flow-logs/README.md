Understanding and Configuring Layered Security in an AWS VPC

Overview

This lab demonstrates how to implement layered security in an AWS VPC using Security Groups and Network ACLs, and how to enable VPC Flow Logs to capture and monitor network traffic for security analysis.

Services Used


Amazon VPC
VPC Flow Logs
Security Groups
Network ACLs
Amazon EC2
Amazon CloudWatch


Steps Performed


Created a custom Amazon VPC with public and private subnets
Configured Security Groups with inbound and outbound rules
Configured Network ACLs as a stateless layer of security
Enabled VPC Flow Logs and directed them to CloudWatch Logs
Generated network traffic and reviewed Flow Log entries
Analysed Flow Log data to identify allowed and denied traffic
