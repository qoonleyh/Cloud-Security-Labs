Check AWS Resources in Trusted Advisor

Overview

This lab demonstrates how to use AWS Trusted Advisor to review security recommendations for AWS resources, identifying potential security risks such as unrestricted security group rules, S3 bucket permissions, and IAM configuration issues.

Services Used


AWS Trusted Advisor
Amazon S3
Amazon VPC (Security Groups)
AWS Identity and Access Management (IAM)


Steps Performed


Created two SSH security groups with open inbound rules
Created two public S3 buckets
Navigated to AWS Trusted Advisor and reviewed the Security category
Identified flagged resources including MFA on Root Account and open security groups
Reviewed investigation recommended items for S3 bucket permissions
Explored Trusted Advisor recommendations and remediation guidance
